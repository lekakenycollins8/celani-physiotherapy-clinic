# Design System

# Design Philosophy

The interface should feel:

- Premium
- Calm
- Professional
- Spacious
- Human-centered
- Modern healthcare focused

The design should prioritize readability, trust and conversion.

The overall experience should resemble premium international rehabilitation and wellness clinics rather than generic hospital websites.

Avoid:

- Generic SaaS aesthetics
- Template-heavy medical layouts
- Corporate stiffness
- Cluttered interfaces
- Overly saturated color palettes
- Excessive gradients
- AI-generic visual patterns

---

# Visual Experience Direction

The UI should combine:

- Scandinavian-inspired minimalism
- Editorial healthcare layouts
- Soft premium elevation
- Spacious composition systems
- Calm interaction design
- Human-centered recovery aesthetics

The website should immediately communicate:

“Professional physiotherapy care you can trust.”

# Typography UX Rules

## Use Strong Whitespace

Typography should breathe.

Avoid:
- Dense text blocks
- Tight paragraph spacing
- Overly compressed layouts

---

## Limit Text Width

Recommended:
- 65–75 characters per line

This improves:
- Readability
- Visual comfort
- Editorial quality

---

## Maintain Strong Contrast

Headings:
- Deep charcoal/navy

Body:
- Soft dark gray

Avoid:
- Pure black
- Low-contrast body text

---

# Responsive Typography Strategy

Typography should scale fluidly across devices.

Prioritize:
- Mobile readability
- Large tap-friendly CTA text
- Clear hierarchy on smaller screens
- Comfortable paragraph spacing

---

# Font Loading Strategy

Use:
- next/font optimization
- Variable fonts where possible
- Font-display swap

Prioritize:
- Performance
- Reduced CLS
- Fast rendering

---

# Spacing System

Use generous spacing throughout.

The layout should feel breathable and calm.

Avoid cramped layouts.

---

## Section Spacing

Desktop:
- 120px–160px vertical spacing

Tablet:
- 96px–120px

Mobile:
- 72px–96px

---

## Card Padding

Recommended:
- 24px–40px

Cards should feel spacious and premium.

---

## Content Width Strategy

Avoid extremely wide text containers.

Recommended:
- 720px–820px reading width
- 1280px max layout width

---

# Container System

## Max Width

Recommended:
- 1280px

Wide layouts should still maintain strong internal spacing.

# Motion Design

Motion should feel:

- Elegant
- Calm
- Intentional
- Soft
- Premium

Animation should support UX rather than distract from it.

---

## Allowed Motion

- Fade-up reveals
- Gentle stagger animations
- Soft hover transitions
- Smooth mobile menu transitions
- Slight image scaling
- Soft opacity transitions

---

## Motion Timing

Recommended:
- 0.4s–0.7s

Easing:
- ease-out
- custom smooth easing

---

## Avoid

- Aggressive parallax
- Excessive movement
- Bouncy animations
- Fast transitions
- Overly playful effects

# Navigation Design

The navigation should feel:

- Minimal
- Spacious
- Premium
- Editorial

---

## Desktop Navbar

Behavior:
- Transparent over hero initially
- Solid background on scroll
- Smooth transition states

Include:
- Services dropdown
- WhatsApp CTA
- Call CTA

---

## Mobile Navigation

Requirements:
- Full-screen premium overlay
- Smooth transitions
- Large touch-friendly links
- Elegant spacing
- Persistent CTAs

Avoid:
- Generic hamburger dropdown patterns
- Cramped mobile menus

---

# Accessibility Direction

Even though WCAG is not the primary focus, maintain:

- Semantic HTML
- Proper contrast ratios
- Accessible buttons
- Keyboard navigation
- Clear focus states
- Proper heading hierarchy

---

# Performance Design Rules

Prioritize:
- Lightweight UI
- Optimized images
- Minimal layout shift
- Fast interaction feedback
- Smooth mobile rendering

Avoid:
- Heavy visual effects
- Unnecessary animation libraries
- Oversized media assets
