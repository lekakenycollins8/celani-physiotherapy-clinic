# MASTER EXECUTION RULES (NON-NEGOTIABLE)

This document overrides all other instructions.

## 1. Single Source of Truth

All implementation decisions MUST come from:

- /docs/design-system.md
- /docs/brand-strategy.md
- /docs/homepage-structure.md
- /docs/services-structure.md
- /docs/seo-strategy.md
- /docs/component-architecture.md
- /docs/development-phases.md
- /docs/mobile-ux.md
- /docs/design-inspirations.md

No external assumptions are allowed unless explicitly required.

---

## 2. No Random Design Decisions

Windsurf MUST NOT:

- Invent new UI patterns
- Introduce new colors
- Change typography system
- Modify spacing rules
- Add unnecessary sections
- Skip defined UX flows

---

## 3. Phase-Based Development Enforcement

Development MUST follow phases strictly:

Phase 1 → Phase 7 in order.

No skipping phases.

No merging phases.

No jumping ahead.

---

## 4. Design System Compliance

Every UI component must strictly follow:

- Typography rules
- Color palette
- Spacing system
- Motion rules
- Border radius rules

Deviation is not allowed.

---

## 5. Component Reuse Rule

If a component exists in:

/components or /sections

It MUST be reused before creating a new one.

---

## 6. SEO Compliance Rule

Every page MUST:

- Use correct heading hierarchy
- Include metadata
- Follow internal linking strategy
- Include local SEO targeting (Nairobi)

---

## 7. Conversion Priority Rule

Every page must prioritize:

1. WhatsApp CTA
2. Call CTA
3. Appointment booking

No page is complete without conversion paths.

---

## 8. Mobile-First Enforcement

All UI must be designed for mobile FIRST.

Desktop is an enhancement, not the base.

---

## 9. Design Consistency Rule

All pages must visually match:

- Design system
- Inspiration references
- Component architecture

No page should feel like a separate product.

---

## 10. Completion Definition

A phase is ONLY complete when:

- UI is implemented
- Responsive behavior works
- SEO rules are applied
- Conversion elements exist
- Design system is followed

## ⚠️ IMPORTANT

Before starting development, always read:

/docs/00-execution-rules.md

This file governs ALL implementation decisions.
